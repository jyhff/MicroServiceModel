# 用户模块功能文档

## 1. 模块概述

| 项目 | 内容 |
|------|------|
| 模块名称 | UserService（用户服务） |
| 功能定位 | 用户注册、登录、等级管理、B币管理、认证管理 |
| 创建时间 | 2026-05-12 |

---

## 2. 功能列表

### 2.1 用户注册

| 功能项 | 说明 |
|--------|------|
| 功能名称 | 用户注册 |
| 功能描述 | 用户注册账号，获取B站账号身份 |
| 输入 | 用户名、密码、邮箱/手机号 |
| 输出 | 用户ID、初始等级LV0、初始B币0 |
| 业务规则 | 用户名唯一、密码强度验证、邮箱/手机号验证 |
| 前置条件 | 无 |
| 后置条件 | 用户账号创建成功 |

---

### 2.2 用户登录

| 功能项 | 说明 |
|--------|------|
| 功能名称 | 用户登录 |
| 功能描述 | 用户登录系统，获取访问令牌 |
| 输入 | 用户名/邮箱/手机号、密码 |
| 输出 | JWT AccessToken、RefreshToken、用户信息 |
| 业务规则 | 密码验证、账号状态检查、生成JWT |
| 前置条件 | 用户账号已注册且未封禁 |
| 后置条件 | 用户登录成功，获得访问令牌 |

---

### 2.3 获取用户信息

| 功能项 | 说明 |
|--------|------|
| 功能名称 | 获取用户信息 |
| 功能描述 | 获取用户详细资料信息 |
| 输入 | 用户ID |
| 输出 | 用户资料（昵称、头像、等级、B币余额、认证信息） |
| 业务规则 | 仅返回公开信息，私密信息需权限验证 |
| 前置条件 | 用户ID有效 |
| 后置条件 | 返回用户信息 |

---

### 2.4 更新用户资料

| 功能项 | 说明 |
|--------|------|
| 功能名称 | 更新用户资料 |
| 功能描述 | 更新用户昵称、头像、签名等资料 |
| 输入 | 用户ID、昵称、头像URL、签名、性别 |
| 输出 | 更新成功确认 |
| 业务规则 | 昵称长度≤50、签名长度≤200 |
| 前置条件 | 用户已登录 |
| 后置条件 | 用户资料更新成功 |

---

### 2.5 等级管理

| 功能项 | 说明 |
|--------|------|
| 功能名称 | 等级管理 |
| 功能描述 | 用户等级系统，LV0-LV6共7个等级 |
| 输入 | 用户行为（观看、点赞、投币等） |
| 输出 | 等级提升、经验值变化 |
| 业务规则 | 每日经验上限100、等级经验阈值固定、LV2解锁高级弹幕 |
| 前置条件 | 用户已登录 |
| 后置条件 | 经验值增加、等级可能提升 |

**等级经验阈值**：
- LV0: 0-100
- LV1: 101-200
- LV2: 201-400
- LV3: 401-600
- LV4: 601-800
- LV5: 801-1000
- LV6: 1001+

---

### 2.6 B币管理

| 功能项 | 说明 |
|--------|------|
| 功能名称 | B币管理 |
| 功能描述 | 用户B币余额管理，用于投币、购买等 |

| 输出 | B币余额变化 |
| 业务规则 | 投币数量无限制（用户需求）、余额实时更新 |
| 前置条件 | 用户已登录 |
| 后置条件 | B币余额变化 |

---

## 4. 业务流程

### 4.1 用户注册流程

```
1. 用户填写注册信息
2. 系统验证用户名唯一性
3. 系统验证密码强度
4. 系统验证邮箱/手机号格式
5. 系统创建用户账号
6. 系统初始化等级LV0、B币余额0
7. 系统返回用户ID和初始信息
```

---

### 4.2 等级提升流程

```
1. 用户完成行为（观看、点赞、投币等）
2. 系统计算获得经验值
3. 系统检查每日经验上限（100）
4. 系统累加用户总经验值
5. 系统检查是否达到下一等级阈值
6. 系统更新用户等级（如需）
7. 系统触发等级变化事件
```

---

### 4.3 投币流程

```
1. 用户选择视频
2. 用户选择投币数量（1或2）
3. 系统检查用户B币余额
4. 系统扣减用户B币
5. 系统增加UP主B币（100%给UP主）
6. 系统记录投币历史
7. 系统增加用户经验值（+10/币）
```
