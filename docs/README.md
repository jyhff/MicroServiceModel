## 🛠️ 技术栈

### 后端
- **框架**: ABP Framework v10.0.2
- **运行环境**: .NET 10.0
- **数据库**: PostgreSQL 15+
- **缓存**: Redis 7
- **消息队列**: RabbitMQ 3
- **搜索引擎**: Elasticsearch 8
- **对象存储**: MinIO / 阿里云OSS
- **实时通信**: SignalR + WebSocket

### 前端
- **框架**: Vue 3 + TypeScript
- **UI组件库**: Ant Design Vue
- **构建工具**: Vite
- **状态管理**: Pinia
- **路由**: Vue Router

### 基础设施
- **容器化**: Docker + Kubernetes
- **API网关**: Ocelot / YARP
- **监控**: Prometheus + Grafana + SkyWalking
- **日志**: ELK Stack (Elasticsearch + Logstash + Kibana)
- **CI/CD**: GitHub Actions / Azure DevOps

## 🚀 快速开始

### 1. 阅读顺序

**新团队成员**：
1. 先读 `01-技术架构设计文档.md` 了解整体架构
2. 再读具体模块的PRD了解功能需求
3. 最后读 `03-接口设计文档.md` 了解API规范

**开发人员**：
1. 先看自己负责的模块PRD
2. 参考 `02-数据库设计文档.md` 建表
3. 按照 `03-接口设计文档.md` 实现API

**DevOps人员**：
1. 先看 `01-技术架构设计文档.md` 了解服务架构
2. 参考 `05-部署运维文档.md` 配置环境

### 2. 文档规范

- 所有PRD文档都包含：功能设计、数据库设计、API设计、技术实现
- 所有代码示例使用实际技术栈（.NET 10.0 + PostgreSQL）
- 数据库表名使用双引号（PostgreSQL标准）
