## 3. 服务拆分原则

### 3.1 拆分策略

| 服务 | 职责 | 拆分依据 | 优先级 |
|------|------|---------|--------|
| **User Service** | 用户注册、认证、画像 | 核心服务，独立演进 | P0 |
| **Video Service** | 视频元数据、管理 | 核心领域 | P0 |
| **Transcode Service** | 视频转码 | 计算密集型，独立扩展 | P0 |
| **Danmaku Service** | 弹幕存储、查询 | 高并发读，独立扩展 | P0 |
| **Danmaku WS** | 实时弹幕推送 | 长连接，独立部署 | P0 |
| **Recommend Service** | 推荐算法 | 计算密集型 | P1 |
| **Search Service** | 全文搜索 | 依赖Elasticsearch | P1 |
| **Live Service** | 直播管理 | 独立业务域 | P1 |
| **Live Push** | 直播推流 | 流媒体服务 | P1 |
| **Interaction Service** | 点赞、投币、收藏 | 高并发写 | P0 |
| **Notification Service** | 消息推送 | 异步处理 | P2 |
| **Admin Service** | 后台管理 | 内部使用 | P2 |

### 3.2 服务间通信

```
同步调用（HTTP/gRPC）：
    API Gateway → User Service
                → Video Service
                → Search Service

异步消息（RabbitMQ）：
    Video Service ──→ Transcode Queue ──→ Transcode Service
    Interaction Service ──→ Notification Queue ──→ Notification Service
    
事件驱动（Kafka）：
    User Behavior ──→ Kafka ──→ Recommend Service
                                 → Analytics Service
```

## 4. 数据架构

### 4.1 数据存储策略

| 数据类型 | 存储方案 | 选型理由 |
|---------|---------|---------|
| **关系数据** | PostgreSQL | 事务支持、成熟稳定 |
| **缓存** | Redis Cluster | 高性能、分布式 |
| **搜索** | Elasticsearch | 全文搜索、聚合分析 |
| **视频文件** | MinIO + CDN | 对象存储、成本低 |
| **日志** | ClickHouse | 时序数据、高效分析 |
| **消息** | RabbitMQ | 可靠消息传递 |
| **大数据** | Kafka + Spark | 流处理、批处理 |

## 5. 技术选型对比

### 5.1 核心框架

| 选型 | 候选方案 | 最终选择 | 理由 |
|------|---------|---------|------|
| 后端框架 | Spring Boot / Django / ABP | **ABP** | 模块化、多租户、成熟生态 |
| .NET版本 | .NET 6 / 7 / 8 / 10 | **.NET 10** | 最新LTS、性能优化 |
| 数据库 | MySQL / PostgreSQL / SQL Server | **PostgreSQL** | JSON支持、扩展性好 |
| 缓存 | Redis / Memcached | **Redis** | 数据结构丰富、持久化 |
| 搜索 | ES / Solr / Meilisearch | **Elasticsearch** | 功能全面、生态成熟 |
| 消息队列 | RabbitMQ / Kafka / RocketMQ | **RabbitMQ** | 可靠性强、易用 |

### 5.2 视频处理

| 组件 | 选型 | 理由 |
|------|------|------|
| 转码引擎 | FFmpeg | 行业标准、功能全面 |
| 流媒体服务器 | SRS / nginx-rtmp | **SRS** | 功能丰富、活跃维护 |
| 播放协议 | HLS / DASH / FLV | **HLS** | 兼容性好、自适应码率 |
| 存储 | MinIO / 阿里云OSS | **MinIO** | 开源、S3兼容 |

## 7. 部署架构

### 7.1 容器化部署

```yaml
# docker-compose.yml (开发环境)
version: '3.8'
services:
  gateway:
    image: bilibili/gateway:latest
    ports:
      - "5000:80"
    depends_on:
      - user-service
      - video-service
      
  user-service:
    image: bilibili/user-service:latest
    environment:
      - ConnectionStrings__Default=Host=postgres;Database=UserDB
    depends_on:
      - postgres
      - redis
      
  video-service:
    image: bilibili/video-service:latest
    environment:
      - ConnectionStrings__Default=Host=postgres;Database=VideoDB
    depends_on:
      - postgres
      - rabbitmq
      
  postgres:
    image: postgres:15
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    volumes:
      - postgres_data:/var/lib/postgresql/data
      
  redis:
    image: redis:7-alpine
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data
      
  rabbitmq:
    image: rabbitmq:3-management
    ports:
      - "15672:15672"  # 管理界面
      
  elasticsearch:
    image: elasticsearch:8.5.0
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
```

### 7.2 Kubernetes部署

```yaml
# user-service-deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: user-service
  namespace: production
spec:
  replicas: 3
  selector:
    matchLabels:
      app: user-service
  template:
    metadata:
      labels:
        app: user-service
    spec:
      containers:
      - name: user-service
        image: registry.example.com/bilibili/user-service:v1.0.0
        ports:
        - containerPort: 80
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /health
            port: 80
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /ready
            port: 80
          initialDelaySeconds: 5
          periodSeconds: 5
      affinity:
        podAntiAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
          - weight: 100
            podAffinityTerm:
              labelSelector:
                matchExpressions:
                - key: app
                  operator: In
                  values:
                  - user-service
              topologyKey: kubernetes.io/hostname
```

## 9. 风险评估

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| 数据库性能瓶颈 | 中 | 高 | 分库分表、读写分离 |
| 缓存雪崩 | 低 | 高 | 多级缓存、熔断降级 |
| 视频转码队列积压 | 中 | 中 | 弹性扩容、优先级队列 |
| 弹幕服务单点故障 | 低 | 高 | 多实例部署、无状态设计 |
| 第三方服务不可用 | 中 | 中 | 熔断降级、本地缓存 |
