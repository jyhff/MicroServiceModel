## 2. API设计规范

### 2.1 URL设计
```
基础URL: https://api.bilibili.com/v1

格式: /{资源}/{操作}

示例:
GET    /api/videos           # 获取视频列表
GET    /api/videos/{id}     # 获取单个视频
POST   /api/videos           # 创建视频
PUT    /api/videos/{id}     # 更新视频
DELETE /api/videos/{id}     # 删除视频
```

### 2.2 HTTP状态码
| 状态码 | 含义 | 使用场景 |
|--------|------|---------|
| 200 | OK | 请求成功 |
| 201 | Created | 资源创建成功 |
| 204 | No Content | 删除成功 |
| 400 | Bad Request | 请求参数错误 |
| 401 | Unauthorized | 未授权 |
| 403 | Forbidden | 禁止访问 |
| 404 | Not Found | 资源不存在 |
| 409 | Conflict | 资源冲突 |
| 422 | Unprocessable | 业务逻辑错误 |
| 429 | Too Many Requests | 请求过于频繁 |
| 500 | Server Error | 服务器错误 |

### 2.3 响应格式
```json
{
  "code": 0,           // 0=成功, 非0=错误码
  "message": "success", // 提示信息
  "data": {},          // 业务数据
  "timestamp": 1704100800000,  // 时间戳
  "requestId": "uuid"  // 请求追踪ID
}
```

### 2.4 分页规范
```json
// 请求
GET /api/videos?page=1&pageSize=20

// 响应
{
  "code": 0,
```

## 5. 错误码规范

### 5.1 通用错误码 (1xxx)
| 错误码 | 说明 |
|--------|------|
| 1001 | 参数错误 |
| 1002 | 未授权 |
| 1003 | 禁止访问 |
| 1004 | 资源不存在 |
| 1005 | 服务器内部错误 |
| 1006 | 服务不可用 |
| 1007 | 请求过于频繁 |

### 5.2 业务错误码 (2xxx)
| 错误码 | 说明 |
|--------|------|
| 2001 | 用户名已存在 |
| 2002 | 手机号已注册 |
| 2003 | 邮箱已注册 |
| 2004 | 用户名或密码错误 |
| 2005 | 账号已被锁定 |
| 2006 | 验证码错误 |

### 5.3 视频错误码 (3xxx)
| 错误码 | 说明 |
|--------|------|
| 3001 | 文件格式不支持 |
| 3002 | 文件大小超出限制 |
| 3003 | 上传会话已过期 |
| 3004 | 转码失败 |
| 3005 | 视频不存在 |

### 5.4 互动错误码 (4xxx)
| 错误码 | 说明 |
|--------|------|
| 4001 | 已点赞 |
| 4002 | 投币已达上限 |
| 4003 | 评论内容过长 |
| 4004 | 发送过于频繁 |
